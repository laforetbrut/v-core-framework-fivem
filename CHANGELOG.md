# Changelog

All notable changes to QBCore Dev Server are documented here.

---

## [1.0.0] - 2026-07-08

### Added (English first)

- **Initial server scaffold** — FXServer artifacts (build 25770), full QBCore framework and default cfx resources installed under `resources/`.
- **Database layer** — `oxmysql` 2.14.1 wired to a local MariaDB instance via `mysql_connection_string`; QBCore schema import prepared.
- **Voice & maps** — `pma-voice`, `qb-radio`, and default hospital/dealer/prison maps included.
- **Development starter** — example resource `resources/[local]/dev-starter` with `/hello` and `/serverinfo` commands demonstrating QBCore client/server APIs.
- **Launchers** — `start.bat` and `start.ps1` to boot the server on port 30120.
- **Project docs** — `README.md`, `RULES.md`, `.gitignore` tailored to a FiveM/QBCore workflow.

### Ajouts (French mirror)

- **Squelette initial du serveur** — binaires FXServer (build 25770), framework QBCore complet et ressources cfx par défaut installés sous `resources/`.
- **Couche base de données** — `oxmysql` 2.14.1 relié à une instance MariaDB locale via `mysql_connection_string` ; import du schéma QBCore préparé.
- **Voix & maps** — `pma-voice`, `qb-radio` et les maps par défaut hôpital/concessionnaire/prison inclus.
- **Ressource de démarrage** — ressource d'exemple `resources/[local]/dev-starter` avec les commandes `/hello` et `/serverinfo` illustrant les API client/serveur de QBCore.
- **Lanceurs** — `start.bat` et `start.ps1` pour démarrer le serveur sur le port 30120.
- **Documentation projet** — `README.md`, `RULES.md`, `.gitignore` adaptés à un workflow FiveM/QBCore.
