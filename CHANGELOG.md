- **Ground drops can no longer be looted from afar (v-inventory, SECURITY)** â opening a ground drop had **no server-side proximity check** while vehicle trunks did, and drop ids are sequential (`drop:1`, `drop:2`, âĶ), so a modified client could enumerate and drain **every drop on the map** remotely. ### Fixed (in-game interface pass)

- **Inventory `openStash` exploit (CRITICAL) + fractional amounts** â the trunk/stash open trusted a client-supplied id, so a modded client could open **any** vehicle's trunk by plate from anywhere and even write arbitrary rows into the `stashes` table. The client now sends the vehicle's **net id**; the server resolves the entity, **checks the player is within 5 m**, and derives the stash id from the **server-read plate**. Ground drops must reference a real drop. Move / drop / give amounts are now **floored** (no fractional stacks), and the stash `kind` is finally sent to the NUI.
- **Drag-drop stacking / placement (inventory + shops)** â dragging a stackable item onto a matching one did not merge, and drops (incl. drag-to-buy in shops) did not land on the targeted slot. Two root causes: oxmysql returns `TINYINT(1)` as a **boolean**, so `stackable == 1` / `usable == 1` silently failed (now coerced to 0/1 on load); and the merge condition tested `not metadata`, but an **empty table `{}` is truthy in Lua**, blocking every merge (now a `noMeta` helper treats nil and `{}` alike). `addToContainer` / `AddItem` now accept a **preferred slot**, so an inventory move and a shop drag-to-buy land where you dropped them (stacking onto a match there, else placing in the empty cell).
- **Spawn fell into the void / showed a default ped** â the default `spawnmanager` revealed a nude freemode ped at the fallback point (often above the ocean) before v-spawn took over, then the GTA switch swooped onto ground that had not streamed yet, so the player fell. The screen is now held **black from the first frame** until a flow takes control; in `switchSpawn` the ped is kept **frozen** at the destination while collision streams and a solid ground Z is found (retried), and is unfrozen **only after** the switch-in completes. The character is also dressed while the screen is black.
- **Inventory drag & drop did nothing in-game** â it used **HTML5 drag-and-drop**, which FiveM's CEF 103 does not fire reliably (it worked in a normal browser, not in-game). Rewritten with **pointer events** (mousedown â a ghost that follows the cursor â mouseup), the technique every mainstream FiveM inventory uses. A plain click no longer triggers a move; equipping by dragging a clothing item onto a body slot still works.
- **Items with no image** â clothing items are inserted with no raster image, so they showed blank. Slots now render a **fallback icon**: a garment glyph for clothing, a generic box for anything else.
- **No mouse cursor in any menu (inventory, admin, banking, shops, clothing)** â the panels rendered but the player could not interact with them. The shared focus helper called `SetNuiFocus` from inside **v-core**, and that native is **scoped to the calling resource**: it returns early when the caller has no NUI frame, and otherwise focuses the caller's own page. `v-core` declares no `ui_page`, so the call was a **silent no-op** â no error, no log line. v-hud (F7) and v-spawn were unaffected because they still called the native locally, which made the bug look module-specific. `v-core/client/focus.lua` now keeps only the reference-counted `nuiOpen` bookkeeping (`MenuOpened` / `MenuClosed` / `IsAnyMenuOpen`), and each owning resource takes and releases focus itself.
- **Premium inventory restyle** â a Quasar-grade visual polish (keeping the orange identity): layered panel gradients + top accent glow + depth shadows, glossier segmented weight bar, framed quick-slots column, richer slots (inner highlight, hover lift + glow, crisper rarity glows, fixed item-image sizing), refined tooltip/context/amount cards and micro-animations. Also: **arms/gloves are now free** in the clothing store.
- **Custom square minimap (v-hud)** â the radar mask is swapped for a square texture (QBCore method, shipped `squaremap.ytd`): the map becomes a clean square and the GTA:O green/blue health & armour bars are **no longer drawn at all**. It's fully **repositionable** â drag it anywhere in layout mode (the player's frame offset shifts the map), no decorative frame in normal mode, aspect-ratio corrected, per-player persistence, and the vanilla mask is restored on resource stop. Hunger & thirst rings are now **always readable** (never fade) and a **one-shot alert fires at 25%** hunger / thirst (amber ring pulse + notification, re-armed above 32%).
